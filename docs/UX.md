# UX-концепция сайта AdsMsk (ads.msk.ru)

Документ описывает пользовательские пути, wireframes ключевых страниц, компонентный состав, CTA-стратегию и структуру форм для обновлённого сайта агентства digital-маркетинга AdsMsk.

Целевая аудитория: предприниматели и менеджеры малого/среднего бизнеса, продающие на маркетплейсах (Wildberries, Ozon), а также владельцы локального и e-commerce бизнеса, которым нужна реклама, SEO, контент и сопровождение.

Основная конверсия — заявка на услугу через один из трёх каналов: Telegram-бот (@UR16_bot), WhatsApp (+7-915-468-39-25), телефон.

---

## 1. User Journey Map (3–4 сценария)

### Сценарий A — «Холодный» переход из поиска/рекламы (новый клиент, конкретная боль)
Пример: продавец на Wildberries ищет «продвижение карточки товара на WB».

1. **Осознание проблемы** — низкие продажи/просмотры карточки, не знает с чего начать.
2. **Поиск** — заходит из Яндекса/Google по запросу услуги → попадает на страницу услуги (`/uslugi/[slug]`), минуя главную.
3. **Оценка релевантности** (5–10 сек) — смотрит Hero: понятен ли оффер, есть ли его сфера (маркетплейсы), есть ли цена/диапазон.
4. **Доверие** — листает блок кейсов/цифр (рост продаж, ROAS, скриншоты статистики WB/Ozon), отзывы.
5. **Понимание процесса** — смотрит «Как мы работаем» (этапы, сроки).
6. **Снятие возражений** — FAQ (сколько стоит, какие гарантии, что нужно от клиента).
7. **Действие (micro-CTA)** — нажимает «Рассчитать стоимость» → попадает в форму/виджет расчёта.
8. **Конверсия (primary CTA)** — либо сразу пишет в Telegram/WhatsApp, либо после расчёта оставляет заявку с параметрами.
9. **Постконверсия** — получает автоответ бота с подтверждением и ожидаемым временем ответа менеджера.

**Ключевые точки риска:** длинный текст без структуры, отсутствие цены/вилки, отсутствие примеров именно по маркетплейсам.

---

### Сценарий B — «Тёплый» переход с главной (сравнивает агентства)
Пример: владелец малого бизнеса собирает 3–5 агентств для сравнения.

1. **Вход на главную** (`/`) — общее впечатление: специализация, для кого, какие услуги.
2. **Сканирование Hero + блока услуг** — понимает спектр (таргет, контекст, SEO, маркетплейсы, контент, сопровождение).
3. **Переход в каталог услуг** (`/uslugi`) — сравнивает, что есть у AdsMsk vs у конкурентов (открытых в других вкладках).
4. **Переход в кейсы** (`/cases`) — ищет кейсы по своей нише/масштабу бизнеса, фильтрует по индустрии или типу услуги.
5. **Переход в «О команде»** (`/komanda`) — проверяет, кто будет делать работу (не просто «менеджер-прокладка»), опыт, фото, должности.
6. **Возврат на главную или в каталог** — повторно проверяет triggers доверия (цифры, логотипы клиентов, сертификаты).
7. **Контакты** (`/kontakty`) — сверяет реальность компании (адрес, телефон, реквизиты, карта).
8. **Primary CTA** — «Получить консультацию» в шапке или в финальном CTA-блоке любой из страниц → переход в Telegram/WhatsApp с заранее заполненным сообщением («Здравствуйте, интересует [услуга]»).

**Ключевые точки риска:** отсутствие фильтрации в кейсах, «пустая» команда (стоковые фото), CTA теряется при долгом скролле (нужен sticky CTA).

---

### Сценарий C — «Горячий» клиент (пришёл по рекомендации/визитке)
Пример: клиента порекомендовали знакомые, он уже знает имя агентства.

1. **Прямой заход** на `/` или сразу в поиске «AdsMsk отзывы» / «AdsMsk кейсы».
2. **Быстрая верификация** — смотрит кейсы и контакты, чтобы убедиться, что это реальная компания.
3. **Сразу совершает primary CTA** — звонит по телефону или пишет в WhatsApp/Telegram без чтения остального контента.

**Вывод для дизайна:** телефон и мессенджеры должны быть видны на 100% страниц в шапке/футере без скролла (sticky header + sticky mobile bar).

---

### Сценарий D — «Возврат» (ретаргетинг / напоминание)
Пример: клиент уже был на сайте, не оставил заявку, вернулся по ретаргетинговому объявлению на страницу услуги.

1. **Вход через рекламное объявление** с конкретным офером/скидкой → попадает на лендинг услуги с UTM.
2. **Фокус на CTA** — пользователь уже «прогрет», ему нужно минимум трения: видит специальное предложение (если есть) и форму расчёта стоимости.
3. **Micro-CTA → Primary CTA** — заполняет калькулятор/мини-форму → получает кнопку «Отправить в Telegram» с предзаполненными данными.
4. **Альтернатива** — если не готов оставить контакты, использует «Скачать прайс/чек-лист» (lead magnet) как промежуточный шаг перед звонком/заявкой.

## 2. Wireframes ключевых страниц

Условные обозначения: блоки идут сверху вниз в порядке появления на странице, ширина — full width контейнера с центрированным контентом (max-width ~1200px), мобильная версия — single column в том же порядке, если не указано иное.

## 4. CTA-стратегия

### Принципы
1. **Множественность каналов на каждом CTA** — везде, где возможно, рядом с primary CTA даём выбор из трёх каналов: Telegram / WhatsApp / Телефон, так как ЦА (предприниматели на маркетплейсах) привыкла именно к мессенджерам, а не к формам.
2. **Минимум трения** — кнопки Telegram/WhatsApp используют deep-link с предзаполненным текстом сообщения (`https://wa.me/79154683925?text=...`, `https://t.me/UR16_bot?start=...`), чтобы пользователю не пришлось формулировать сообщение самому.
3. **Контекстный текст CTA** — текст кнопки адаптируется под страницу/услугу (не просто «Отправить», а «Получить консультацию по продвижению на WB»).
4. **Sticky-доступность** — на мобильных везде закреплена нижняя панель с иконками трёх каналов; на десктопе — телефон и кнопка «Получить консультацию» в шапке.
5. **Лестница вовлечения** — micro-CTA (низкое обязательство: «Рассчитать стоимость», «Скачать чек-лист») → primary CTA (среднее обязательство: «Получить консультацию») → прямой контакт (Telegram/WhatsApp/звонок).

### Иерархия приоритетов CTA
1. **Primary — «Получить консультацию»**: главный конверсионный элемент, доступен на каждой странице минимум 2 раза (header + финальный блок), на длинных страницах услуг — дополнительно как sticky-бар.
2. **Secondary — «Смотреть кейсы»**: используется там, где пользователь ещё не готов оставить контакт, нужен доп. аргумент доверия.
3. **Micro — «Рассчитать стоимость» / «Скачать чек-лист»**: снижает порог входа для холодного трафика, формирует лид через промежуточное действие, не требующее звонка/чата.

## 5. Структура форм обратной связи

### 5.1. Финальная форма (используется в конце страниц: главная, каталог, кейсы, команда, контакты)

Минималистичная форма для снижения отказов:

| Поле | Тип | Обязательное | Комментарий |
|---|---|---|---|
| Имя | text | да | placeholder «Как к вам обращаться» |
| Телефон или Telegram/WhatsApp | tel/text | да | единое поле с подсказкой «удобный способ связи» |
| Интересующая услуга | select | нет | список услуг + «Не знаю, нужна консультация» |
| Комментарий | textarea | нет | необязательное поле, placeholder «Кратко опишите задачу» |
| Согласие на обработку ПДн | checkbox | да | ссылка на политику конфиденциальности |
| Кнопка отправки | button | – | текст контекстный (см. CTA-матрицу) |

После отправки: экран благодарности + альтернатива «или напишите нам напрямую» с тремя иконками-ссылками (TG/WA/телефон), на случай если форма не устроит пользователя по скорости ответа.

### 5.2. Калькулятор стоимости (страница услуги, micro-CTA)

Упрощённый пошаговый виджет (2–3 шага), без обязательной авторизации:

**Шаг 1 — Параметры задачи** (зависят от услуги, примеры для маркетплейсов):
- Маркетплейс: WB / Ozon / Я.Маркет / несколько
- Количество карточек товара: число/слайдер
- Текущий статус: «Только выходим» / «Уже продаём, нужен рост»

**Шаг 2 — Бюджет/объём**:
- Желаемый бюджет на продвижение: select-диапазоны («до 30 000 ₽», «30–100 000 ₽», «свыше 100 000 ₽»)
- Срок: «Как можно скорее» / «В течение месяца» / «Просто изучаю варианты»

**Шаг 3 — Результат и контакт**:
- Показ примерной вилки стоимости («от ___ до ___ ₽/мес»)
- Поле: Telegram/WhatsApp/телефон
- Кнопка: «Отправить расчёт менеджеру» → данные расчёта автоматически передаются в Telegram-бот (@UR16_bot) вместе с контактом клиента, чтобы менеджер видел контекст заявки.

### 5.3. Контактная форма на странице `/kontakty` (основная форма страницы)

Более подробная, так как пользователь уже целенаправленно пришёл оставить заявку:

| Поле | Тип | Обязательное |
|---|---|---|
| Имя | text | да |
| Компания/проект (необязательно) | text | нет |
| Телефон | tel | да |
| Telegram или WhatsApp (никнейм/номер) | text | нет |
| Интересующая услуга | select (мультивыбор) | нет |
| Сообщение | textarea | нет |
| Удобное время для звонка | select («Сейчас» / «Утро» / «День» / «Вечер») | нет |
| Согласие на обработку ПДн | checkbox | да |

### 5.4. Общие требования к формам
- Валидация в реальном времени (подсветка ошибок без перезагрузки).
- На мобильных устройствах — крупные поля ввода (мин. 44px высота тапа), числовая клавиатура для телефона.
- После отправки — событие в Яндекс.Метрику (цель «lead_form_submit») для каждого типа формы отдельно (финальная форма, калькулятор, контактная форма) — это нужно для корректной аналитики эффективности CTA по страницам.
- Все формы дублируют сообщение в Telegram-бот (@UR16_bot) как основной канал обработки заявок менеджером.

## 6. Резюме приоритетов для разработки

1. Header с постоянной видимостью CTA и контактов (sticky) + mobile sticky bar — внедрить первым, так как влияет на конверсию сценария C («горячие» клиенты).
2. Страница услуги (`/uslugi/[slug]`) — приоритетный шаблон, так как принимает основной SEO/рекламный трафик (сценарий A); должна быть самодостаточной без необходимости уходить на другие страницы.
3. Калькулятор стоимости — снижает порог входа для холодного трафика, реализуется как переиспользуемый компонент на всех страницах услуг.
4. Кейсы с фильтрацией — критичны для сценария B (сравнение агентств).
5. Финальный CTA-блок с формой — единый переиспользуемый компонент на всех страницах для консистентности и упрощения разработки.
